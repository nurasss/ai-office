# AI Office glossary

## Product terms

| Term | Approved meaning |
|---|---|
| AI Office | Digital office made of seven specialized AI employees |
| PMO | Orchestrator and task-routing agent |
| Agent | Specialized AI employee with a role, prompt, tools and RAG namespace |
| RAG namespace | Isolated knowledge space used for retrieval |
| common_corporate | Shared corporate context available to every agent |
| verified input | Compact checked result passed by PMO from one agent to another |
| human review | Required human check before publication, deployment or finance approval |
| route-only mode | API mode that shows routing and RAG hits without calling an LLM |

## Preferred wording

- Use "digital office" or "AI employees" for product explanation.
- Use "first draft" when output still needs human editing.
- Use "verified check" when the agent performed a constrained review.
- Use "needs_review" for incomplete accounting or compliance cases.

## Avoid

- Do not call the system a fully autonomous replacement for employees.
- Do not imply finance, legal or deployment decisions happen without approval.
- Do not invent model prices, tax rates, benchmarks or external market numbers.
